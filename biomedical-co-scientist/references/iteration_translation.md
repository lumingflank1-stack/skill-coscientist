# Iteration, Validation, and Translation

Use this reference when the user gives analysis results, asks what the results mean, wants new hypotheses from data, or needs experiments, translation, manuscript, or grant framing.

## Result Interpretation

Classify each result:

- Strong support
- Moderate support
- Weak support
- Neutral
- Contradictory
- Not interpretable

For each classification, state:

- What can be claimed
- What cannot be claimed
- Main alternative explanation
- Next analysis
- Next experiment
- How the hypothesis should change

## Revision Rules

After analysis:

- Keep a hypothesis if the central prediction is supported by sample-level or perturbation evidence.
- Downgrade a hypothesis if support is cell-level only, confounded, or nonspecific.
- Merge hypotheses if they share the same upstream trigger and downstream readout.
- Split a hypothesis if one branch is supported and another branch is contradicted.
- Reject a hypothesis only when a decisive, well-controlled result contradicts the core prediction.
- Generate second-level hypotheses from unexpected results, but label them as new candidates.

## Translational Strategy

For each intervention point:

- Target node
- Biological rationale
- Tool compounds or genetic perturbations
- Clinically realistic compounds, if any
- Metabolites, dietary molecules, or microbial products, if relevant
- Perturbation database to query, such as LINCS/CMap, Connectivity Map, DepMap, ChEMBL, DrugBank, or PubChem when appropriate
- Biomarker readouts
- Safety concerns
- Go/no-go criteria

Separate "screening tool" from "clinically plausible therapeutic candidate".

## Manuscript or Grant Framing

Convert a prioritized mechanism into:

- Working title
- Central hypothesis
- One-sentence significance
- Innovation
- Figure plan
- Specific aims or result sections
- Key methods
- Decisive validation
- Risk and alternative strategy
- Target journals or funding angle

Avoid writing a story that is stronger than the evidence. State validation gaps plainly.
